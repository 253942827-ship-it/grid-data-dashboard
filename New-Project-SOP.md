# 新项目创建 SOP (Standard Operating Procedure)

> 本文件定义在本 Workspace 中创建任何一个新项目的标准化操作流程。
> **执行新项目前，必须先阅读本文档并严格按步骤执行。**
> **所有项目文件必须存放在对应的项目文件夹中，严禁存放到电脑的其他位置。**

---

## SOP 流程概述

1. **前置检查** — 确认全局工作台、踩坑日志和本 SOP 均已阅读
2. **创建项目文件夹** — 按规范创建 `project-{编号}-{名称}/` 目录
3. **初始化项目** — 搭建项目基础结构和配置文件
4. **确认文件归位** — 所有文件均在项目文件夹内
5. **启动开发** — 开始执行具体任务

---

## 步骤详解

### 步骤 1：前置检查

在开始任何新项目之前，必须先完成以下操作：

1. 阅读 [全局工作台](Global-Workbench.md)，确认工作规则
2. 阅读 [全局复利与踩坑日志](Global-Compound-and-Pitfall-Log.md)，回顾过往经验教训
3. 阅读本 SOP 文档，确认流程无误

**确认三份文档均阅读完成后**，方可进入下一步。

### 步骤 2：创建项目文件夹

1. 确定当前项目的下一个可用编号
   - 查看 `Workspace/projects/` 目录下已有的所有 `project-*` 文件夹
   - 找到最大编号 +1 即为新项目编号
2. 按命名规范创建文件夹：
   ```
   Workspace/projects/project-{三位数编号}-{项目英文名称}/
   ```
3. 在项目文件夹内创建基础目录结构（根据项目类型选择）：

   **常规项目结构：**
   ```
   project-{编号}-{名称}/
   ├── README.md           # 项目说明（包含项目名称、编号、目标简介、创建日期）
   ├── docs/               # 项目文档
   ├── src/                # 源代码
   ├── assets/             # 资源文件（图片、字体、excel数据等）
   └── outputs/            # 交付物输出
   ```

   **前端/DOM 项目结构：**
   ```
   project-{编号}-{名称}/
   ├── README.md
   ├── docs/
   │   └── dashboard.html   # 生成的看板文件
   ├── src/
   │   └── generate.py       # 生成脚本
   ├── assets/               # 数据文件、图片等
   └── outputs/              # 最终交付物
   ```

   **数据分析项目结构：**
   ```
   project-{编号}-{名称}/
   ├── README.md
   ├── data/                 # 原始数据文件（Excel、CSV等）
   ├── src/
   │   └── analysis.py       # 分析脚本
   ├── docs/
   │   └── report.html       # 分析报告
   └── outputs/
   ```

### 步骤 3：初始化项目

1. 如果是编程项目，根据语言/框架需要初始化配置文件：
   - Node.js: `package.json`
   - Python: `requirements.txt`
2. 如果项目需要 git 管理，在项目文件夹内独立初始化 git 仓库
3. 在 `README.md` 中记录项目的基本信息，至少包含：
   - 项目编号和名称
   - 创建日期
   - 项目目标描述
   - 数据结构说明（如涉及数据）

### 步骤 4：确认文件归位

**这是强制检查步骤，不可跳过：**

- [ ] ✅ 项目文件夹已创建在 `Workspace/projects/` 下
- [ ] ✅ 项目编号正确（取最大编号 +1）
- [ ] ✅ 项目文件夹内包含 `README.md`
- [ ] ✅ 所有与该项目相关的文件均已移入项目文件夹
- [ ] ✅ 无项目文件遗留在桌面、Downloads、Documents 或其他位置
- [ ] ✅ 临时分析文件存放在项目内的 `work/` 目录

### 步骤 5：启动开发

1. 所有工作产物（代码、文档、设计稿）均保存在项目文件夹内
2. 临时文件（如分析数据、草稿）可放在项目文件夹下的 `work/` 目录
3. 重要交付物放在项目文件夹下的 `outputs/` 目录
4. 数据文件放在项目文件夹下的 `assets/` 或 `data/` 目录
5. **严禁将项目文件存放在项目文件夹之外的位置**

---

## 项目文件夹结构速查

| 目录/文件 | 用途 | 是否必须 |
|-----------|------|----------|
| `README.md` | 项目说明 | ✅ 必须 |
| `src/` | 源代码 | 建议 |
| `docs/` | 项目文档、报告 | 建议 |
| `assets/` | 资源文件 | 可选 |
| `data/` | 原始数据 | 可选 |
| `outputs/` | 交付物 | 建议 |
| `work/` | 临时工作文件 | 可选 |

- [ ] ✅ 已阅读本 SOP
- [ ] ✅ 已确定项目编号（取 projects/ 下最大编号 +1）
- [ ] ✅ 已创建 `projects/project-{编号}-{名称}/` 目录
- [ ] ✅ 已在项目目录下创建 `README.md`
- [ ] ✅ 已建立项目的基础目录结构（src/、docs/ 等）
- [ ] ✅ 所有项目文件均存放在项目文件夹内，无零散文件遗留
