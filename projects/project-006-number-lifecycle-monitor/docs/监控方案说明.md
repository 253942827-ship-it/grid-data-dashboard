# 号码生命周期监控方案

## 1. 时间设定规则

```
入网当月 = T 月
T+1  = 入网次月
T+2  = 入网后第2个月
...  
T+6  = 入网后第6个月
```

**各周期对应监控内容：**

| 周期 | 监控内容 | 说明 |
|------|----------|------|
| T | 📊 出账监控 | 入网当月出账状态 |
| T+1 ~ T+6 | 💰 佣金回款 | 逐月佣金回款跟踪 |
| T+1 ~ T+6 | 📈 状态监控 | 逐月号码状态跟踪 |

**示例：** 入网日期 2026年4月 → T = 2026年4月

| 周期 | 对应月份 | 监控项目 |
|------|----------|----------|
| T | 2026年4月 | 出账 |
| T+1 | 2026年5月 | 出账 + 状态 + 佣金 |
| T+2 | 2026年6月 | 出账 + 状态 + 佣金 |
| T+3 | 2026年7月 | 出账 + 状态 + 佣金 |
| T+4 | 2026年8月 | 出账 + 状态 + 佣金 |
| T+5 | 2026年9月 | 出账 + 状态 + 佣金 |
| T+6 | 2026年10月 | 出账 + 状态 + 佣金 |

## 2. 三个监控板块

### 板块一：📊 出账监控（T ~ T+6）
- 每个号码在 T ~ T+6 各月的出账状态
- 出账金额统计
- 未出账 / 异常预警

### 板块二：📈 状态监控（T+1 ~ T+6）
- 每个号码在各月的状态
- 正常 / 停机 / 销户 / 其他
- 状态变更跟踪

### 板块三：💰 佣金回款监控（T+1 ~ T+6）
- 每月佣金应收金额
- 每月实收金额
- 回款进度（T+1 到 T+6 各月累计）
- 逾期预警

## 3. 需要你提供的数据格式

请按以下格式提供数据，放在 `data/` 目录下：

### ① 状态数据（Excel）
| 列 | 说明 | 示例 |
|----|------|------|
| 号码 | 11位手机号 | 18925181256 |
| 月份 | 数据所属月份 | 2026-07 |
| 状态 | 正常/停机/销户/其他 | 正常 |

### ② 佣金数据（Excel）
| 列 | 说明 | 示例 |
|----|------|------|
| 号码 | 11位手机号 | 18925181256 |
| 月份 | 佣金所属月份 | 2026-07 |
| 应收金额 | 该月应回款额 | 50.00 |
| 实收金额 | 该月已回款额 | 50.00 |
